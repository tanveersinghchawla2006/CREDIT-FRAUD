Credit Card Fraud Detection

"Can machine learning models trained on anonymized transaction patterns reliably detect fraudulent credit card activity — and how do Random Forest and XGBoost compare in accuracy, recall, and real-time fraud detection performance?"

Problem Statement

Credit card fraud causes over $32 billion in losses globally each year. Fraudulent transactions — including card cloning, stolen card details used online, and behavioral anomalies — are rare events buried within millions of legitimate transactions. Existing rule-based bank systems generate excessive false alarms and miss sophisticated fraud patterns. A machine learning model capable of detecting complex fraud in real time, with high recall and low false negatives, is critically needed by the financial industry.

Literature Review / Market Research

Current fraud detection systems at Visa and Mastercard use classical gradient boosting and deep learning. The ULB benchmark dataset (2013) is widely studied — top published solutions achieve ROC-AUC of 0.97–0.98 using XGBoost and Random Forest. Class imbalance (0.17% fraud rate) remains a key challenge. No commercial bank has deployed quantum ML for real-time fraud detection as of 2026.

Research Gap / Innovation

This project builds an optimized classical ML pipeline using Random Forest and XGBoost, tackling the key challenge of extreme class imbalance (0.17% fraud rate) via SMOTE-based oversampling. The focus is on maximizing Recall to minimize missed fraud, while controlling false positives — a balance that rule-based systems consistently fail to achieve. Comparative evaluation of both models on standardized metrics provides clear, deployable insights for real-world fraud detection.

System Methodology

Dataset / Input

ULB Credit Card Fraud Dataset — 284,807 real European transactions (Sept 2013). Features: Time, Amount, V1–V28 (PCA-anonymized). Target: Class (0 = Genuine, 1 = Fraud). Only 0.17% fraud. Class imbalance corrected using SMOTE — synthetic fraud samples generated from 492 to 227,451.

Model / Architecture

Pipeline: StandardScaler (Amount, Time) → SMOTE (minority class oversampling) → 80/20 stratified train-test split → Random Forest (100 trees, max_depth=10, class_weight='balanced') → XGBoost (100 trees, lr=0.1, scale_pos_weight adjusted) → Evaluation: Precision, Recall, F1-Score, ROC-AUC, Confusion Matrix.

Results & Analysis

Best ROC-AUC Score 0.98

Random Forest achieved Recall of 88% and ROC-AUC of 0.98, outperforming XGBoost (Recall 87%, AUC 0.97). Out of 98 real fraud cases in the test set, 86 were correctly flagged — only 12 missed. False alarm rate was minimal: 115 false positives out of 56,864 genuine transactions. Both models significantly outperform the baseline rule-based system benchmark of ~70% detection rate.
